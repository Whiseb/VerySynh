# Contributing to VerySynh

VerySynh is an open architecture project. Contributions are welcome within the terms of the license.

## This project is currently an architectural specification

There is no production code yet. The goal of this stage is to validate and improve the architecture.

## How to contribute

- **Discuss the architecture.** Open an Issue with your thoughts, questions, or criticism.
- **Propose improvements.** The `docs/` folder contains the full specification. If you see a gap or a flaw, open a Pull Request or start a Discussion.
- **Help with formalization.** `docs/theory.md` contains the formal model. Mathematical rigour is especially welcome.
- **Plan the prototype.** `prototype/` is currently a placeholder. Ideas about implementation, technology stack, and architecture are welcome.

## Code of Conduct

- Be respectful.
- Critique the architecture, not the author.
- Assume good faith.
